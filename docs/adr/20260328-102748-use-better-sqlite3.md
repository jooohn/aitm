# Use better-sqlite3 as the SQLite client

**Date:** 2026-03-28
**Status:** accepted

## Context

The repository management feature requires a local SQLite database at `~/.aitm/aitm.db`. A Node.js SQLite client was needed for the Next.js backend.

## Decision

Use `better-sqlite3` as the SQLite client library.

## Consequences

- All database calls are synchronous, which fits naturally with Next.js server-side code and avoids async complexity for simple CRUD operations.
- Native Node.js addon — requires a build step on install, but prebuilt binaries cover common platforms.
- Not suitable for edge runtimes (Vercel Edge, Cloudflare Workers); if aitm ever targets those, the DB layer would need to be replaced.

## Alternatives considered

- **`sql.js`** — pure JavaScript (WASM), no native build, but runs entirely in memory and requires manual persistence.
- **`drizzle-orm` + `better-sqlite3`** — adds a type-safe query builder, but the schema is simple enough that raw prepared statements are sufficient.
- **`@libsql/client` (libSQL/Turso)** — supports both local files and remote Turso databases, but adds unnecessary complexity for a local-only tool.
